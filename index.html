Lernziele

  • Verstehen, wie Copilot arbeitet
  • Copilot sicher und produktiv einsetzen
  • Praktische Übungen: Setup, TDD, Agent‑Modus
  • Bewerten von Vorschlägen und Sicherheitsaspekten
Betonung: Ziel ist nicht, Copilot blind zu vertrauen, sondern ein Werkzeug effizient zu nutzen.

09:30 — 10:15: Einführung & Funktionsweise

Was ist Copilot?

  • KI‑gestützter Codeassistent
  • Kontextbasierte Vorschläge
  • Integration in IDEs / Codespaces
Kurzer technischer Überblick (Modelle, Trainingsdaten, Wahrscheinlichkeitscharakter). Auf Missverständnisse hinweisen: Vorschläge sind probabilistisch.

Setup & Konfiguration

  • Extension installieren (VS Code / JetBrains)
  • Anmeldung, Aktivierung
  • Einstellungen: Vorschlagsdichte, Dateitypen
Live‑Demo: schnell Copilot in VS Code zeigen (falls möglich). Alternativ Screenshots. Zeig kurz die wichtigsten Einstellungen.

Hands‑on 1 (10 Minuten)

Erste Begegnung: Fibonacci

  1. Neue Datei in Python/JS
  2. Kommentar: "Implement a function that returns the nth Fibonacci number"
  3. Akzeptiere/ablehne Vorschläge, notiere 3 gute & 3 schlechte
Die Teilnehmer sollen lokal oder im Codespace arbeiten. Ziel: Gefühl für Qualität. Sammle ein paar Beispiele im Chat/auf dem Whiteboard.

11:30 — 12:15: Best Practices

  • Tests zuerst (TDD) vs. generieren lassen
  • Code‑Review & Tests sind Pflicht
  • Linters, Pre‑commit Hooks
Praxisbeispiele: wann Vorschläge anpassen, wann verwerfen. Erinnerung an Lizenz- und Sicherheitsprüfungen.

Mittagspause

60 Minuten Pause. Teilnehmer bitte pünktlich zurück.

13:15 — 14:30: Agent‑Modus & Modularisierung

Agenten nutzen, um Boilerplate/Module zu erzeugen

Erkläre Konzept Agent: wiederholbare Prompt‑Workflows, Generatoren für Module, Risiken bei automatischen Änderungen.

Hands‑on 2 (40 Minuten)

Aufgabe: User Service (Express / Flask)

  1. Speziﬁkation schreiben (README-style)
  2. Agent / Copilot verwenden, um Dateien zu erzeugen
  3. Implementierung ergänzen, Tests schreiben (Jest / pytest)
  4. Refactor & Review
Zeitbox beachten: 10 Min Specs, 20 Min Coding + Tests, 10 Min Review. Moderatoren: bei Bedarf Hilfestellung geben.

14:30 — 15:00: Case Studies & Diskussion

  • Effizienzsteigerungen
  • Auswirkungen auf Zusammenarbeit & Ownership
  • Lizenzfragen & Community‑Effekte
Beispiele aus Lessons: CRUD-Boilerplate, TDD-Workflow. Öffne Diskussion: Wie ändert Copilot eure Review-Prozesse?

15:15 — 16:00: Abschlussübungen & Review

Exercise: Validierung & Sicherheit (30 min)

  1. Identifiziere mögliche Unsicherheiten in generiertem Code
  2. Schreibe Tests / Hinweiskommentare
  3. Erstelle ein kleines Policy‑Dokument: "Do/Don't when using Copilot"
Teilnehmer präsentieren kurz (2‑3 Minuten) ihre Ergebnisse. Stelle Leitfragen: Datenlecks, Geheimnisse, Abhängigkeiten, Lizenzchaos.

Bitte Feedback-Formular: Was war hilfreich, was fehlt? Biete Follow‑up‑Session an (z. B. vertiefte TDD-Session).
